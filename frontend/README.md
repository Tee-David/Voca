# Voca — Frontend

Next.js 16 (App Router) + TypeScript + Tailwind v4 + shadcn/ui. Hosts the whole user-facing app plus every API route (auth, uploads, library CRUD, admin).

## Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 16 (App Router, Turbopack) |
| Language | TypeScript 5 |
| Styling | Tailwind v4 + `tw-animate-css` |
| Components | shadcn/ui + Base UI |
| Animation | Framer Motion + GSAP |
| State | Zustand + TanStack Query |
| Auth | NextAuth.js v5 beta (credentials + JWT) |
| ORM | Prisma 5 + CockroachDB |
| Storage | Cloudflare R2 via `@aws-sdk/client-s3` |
| TTS (browser) | `kokoro-js` in a Web Worker |
| PDF | `pdfjs-dist` |
| EPUB / DOCX | `epubjs` + `mammoth` |
| PWA | `@serwist/next` |

## Scripts

```bash
npm run dev        # start dev server (Turbopack) at :3000
npm run build      # prisma generate + next build
npm run start      # serve the production build
npm run db:push    # push prisma/schema.prisma to CockroachDB
npm run db:seed    # run prisma/seed.ts (creates admin user)
```

## Database

Schema: [prisma/schema.prisma](prisma/schema.prisma)

Models: `User`, `UserPreferences`, `Session`, `PasswordResetToken`, `Book`, `ReadingProgress`, `Bookmark`, `Audiobook`, `ReadingSession`.

All per-user queries use compound indexes (`[userId, uploadedAt]`, `[userId, lastReadAt]`, `[userId, percentComplete]`, …) so the library grid and dashboard stay fast as the data grows.

## Admin

The email `wedigcreativity@gmail.com` is the single admin. The check lives in the API routes under `src/app/api/admin/**` and in `src/app/(app)/settings/page.tsx` (Users tab visibility). Change the constant `ADMIN_EMAIL` in those files to swap admins.

## PWA

Service worker is generated by `@serwist/next`. Manifest lives at [public/manifest.json](public/manifest.json). The custom full-screen splash is [public/voca_preloader.html](public/voca_preloader.html), mounted as a React component at `src/components/brand/VocaPreloader.tsx`.
